# Ассистент для редактора

Одностраничное веб-приложение для проверки входящего запроса от клиента (брифа): насколько он полон, где противоречия, какие вопросы задать до старта работы.

## О проекте

Проект сделан как статический MVP для редакторов: интерфейс и основная логика находятся в одном файле `Assistant.html`, без сборки.  
Для AI-анализа используется Netlify Function (`netlify/functions/analyze-brief.js`), которая безопасно обращается к OpenAI через `OPENAI_API_KEY` из окружения. Приложение сохраняет рабочее состояние в `localStorage` (`editorassistant.v1`) и помогает пройти путь «ввод запроса → разбор → уточнения → понимание задачи».

### Стек и принципы

- **Frontend**: HTML + CSS + Vanilla JS (single-file в `Assistant.html`).
- **Serverless**: Netlify Function `analyze-brief` для проксирования AI-запроса.
- **Хостинг**: Netlify (`netlify.toml`, редирект `/` на `Assistant.html`).
- **Данные**: локальное хранение только состояния интерфейса в `localStorage`.
- **AI-анализ**: OpenAI Responses API через серверную функцию (ключ не попадает во фронтенд).
- **Вспомогательный CLI-скрипт**: `ask_ai.py` (Python + `python-dotenv` + `openai`) — не участвует в основном пользовательском сценарии.

## Как открыть

Скачайте репозиторий или клонируйте его, затем откройте файл **`Assistant.html`** в браузере (двойной клик или «Открыть с помощью…» → браузер).

Отдельный сервер и установка зависимостей **не нужны**.

## Что делает ассистент

- Разбивает запрос по блокам: контекст, проблема, задача, решение, конечный результат, сроки.
- Подсвечивает пробелы и формулирует уточняющие вопросы.
- Ищет типовые противоречия (например, разный объём или тон в одном брифе).
- Собирает черновик «понимания задачи» для копирования в переписку или бриф.

Логика разбора работает в двух режимах:
- **AI-режим (при наличии API-ключа)**: отправляет текст во внешний API, получает структурированный JSON (противоречия, вопросы, сигналы по категориям, короткое понимание задачи).
- **Fallback-режим**: если AI недоступен или ответ невалидный, автоматически включаются локальные эвристики.

## Настройка AI-режима через .env / Netlify

1. В Netlify откройте: **Site configuration → Environment variables**.
2. Добавьте переменную `OPENAI_API_KEY` со своим ключом OpenAI.
3. Сохраните и запустите redeploy сайта.
4. В интерфейсе приложения просто нажмите **Разобрать запрос** — фронтенд вызывает `/.netlify/functions/analyze-brief`.
5. Если функция недоступна или ответ AI невалидный, приложение автоматически переключится на эвристический fallback.

### Локальная проверка функции

1. Создайте локальный `.env` в корне проекта и добавьте:
   - `OPENAI_API_KEY=...`
2. Запустите Netlify Dev:
   - `netlify dev`
3. Откройте локальный URL из Netlify Dev и проверьте сценарий кнопки **Разобрать запрос**.

## Важно про приватность

- В AI-режиме текст брифа отправляется во внешний сервис через вашу Netlify Function.
- Не используйте AI-режим для чувствительных данных, если политика вашей команды это запрещает.
- Не храните ключи в репозитории и не коммитьте `.env`/секреты.

## Публикация на Netlify (автообновление после мёржа в `main`)

Сайт можно привязать к этому репозиторию: после каждого **push в ветку production** (ниже — `main`) Netlify заново выкладывает файлы. Обычный сценарий: **merge pull request в `main`** → на GitHub попадает новый коммит → Netlify сам подхватывает изменения.

### Один раз: подключить репозиторий

1. Зайди на [Netlify](https://www.netlify.com/) и войди в аккаунт (или зарегистрируйся).
2. **Add new site** → **Import an existing project**.
3. **Deploy with GitHub** → разреши Netlify доступ к GitHub (если спросит — выбери репозиторий **`ivan-smirnov/editorassistant`** или «All repositories»).
4. В настройках деплоя:
   - **Branch to deploy:** `main`
   - **Build command:** оставь **пустым** (в репозитории есть `netlify.toml`, команда сборки не нужна).
   - **Publish directory:** `.` или оставь как подставит Netlify из `netlify.toml` — корень репозитория.
5. Нажми **Deploy**. Через минуту появится ссылка вида `https://что-то.netlify.app`.

Файл **`netlify.toml`** уже задаёт папку публикации и перенаправление с `/` на `Assistant.html`, чтобы главная страница сайта открывала приложение.

### Автодеплой при мёрже

- В **Site configuration → Build & deploy → Continuous deployment** проверь, что включён деплой из **GitHub** и production-ветка — **`main`**.
- Любой **merge в `main`** (или прямой push в `main`) запускает новый деплой; менять что-то в Netlify для каждого мёржа не нужно.

### Полезно

- **Deploy previews:** для pull request’ов Netlify по желанию собирает превью-ссылки (в настройках сайта / на GitHub можно включить проверки).
- Кастомный домен: **Domain management** в панели сайта.

## Ограничения MVP

- Нет учётной записи и синхронизации между устройствами.
- Анализ не заменяет живое обсуждение с клиентом и не понимает смысл так же глубоко, как большая языковая модель.
